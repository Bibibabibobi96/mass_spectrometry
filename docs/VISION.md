# 自然语言驱动的质谱生成式工程愿景

本文是仓库长期使命、能力边界和目标形态的唯一权威。它回答“最终建设什么”，不记录当前项目状态、
实施顺序或具体参数基线。当前事实仍以各项目`docs/PROJECT.md`为准，阶段顺序见
[`ROADMAP.md`](ROADMAP.md)，执行与归档规则见仓库[`README.md`](../README.md)。

## 使命

建设一个自然语言驱动、物理合同约束、可验证且可追溯的质谱生成式工程平台。使用者可以给出性能目标、
几何限制、接口条件和使用场景；系统将其转化为机器可执行的工程规格，自动完成候选设计、独立物理模拟、
诊断与优化，并在满足验收条件后发布正式模拟结果、求解器模型、CAD和验证报告。

平台面向质谱仪器及离子/电子光学部件，不以某个求解器为中心。COMSOL、SIMION、MATLAB、Python和
SolidWorks是受统一参数、接口与验收合同约束的平等实现工具。

## 目标闭环

```text
自然语言需求
  → 结构化工程规格与验收指标
  → 设计族和现有设计选择，或新设计空间生成
  → 参数化几何、物理场、粒子源和接口合同
  → 多保真度模拟与跨求解器验证
  → 误差归因、参数优化和结构优化
  → 候选验收与失败回流
  → 正式模型、CAD、结果、参数和可复现报告
```

“自动”不等于跳过物理定义或门禁。需求中缺失的单位、边界条件、容差和优先级必须被显式补齐或标记为
未决；没有通过适用验证的候选不得因优化器给出较高分数而转正。

## 核心能力

### 需求到规格

- 从自然语言提取目标质量、电荷态、分辨率、传输率、尺寸、电压、压力、制造和接口约束。
- 区分硬约束、优化目标、诊断量与参考示例，生成可审阅的机器规格。
- 在运行前识别目标冲突、信息缺口和不可验证表述，避免把猜测隐藏在求解器脚本中。

### 参数化设计与设计选择

- 从已有正式设计实例化新工况，或在没有合适设计时创建受控候选设计空间。
- 联合描述几何、材料、电极、电压、粒子源、气体、时间控制、机械接口和制造限制。
- 保持`baseline → resolved → solver/CAD`单向派生，优化变量只能通过候选合同进入设计。

### 模拟、诊断与优化

- 按任务风险选择解析模型、理想场、局部分量替换、低成本数值模型和正式三维模型等不同保真度。
- 自动调用独立求解器，统一事件、粒子、场分量、统计和坐标语义，不用共享同一数值场伪造闭合。
- 根据灵敏度、误差归因和功能指标调整结构与运行参数，包括电极尺寸、间距、屏蔽结构、电压和接口。
- 同时处理性能、鲁棒性、可制造性、计算成本和部件集成，不把单一峰值指标当作完整设计质量。

### 验证与正式化

- 为每类功能建立明确验收合同，例如分辨率、传输率、质量选择性、碰撞统计、束斑、发散、能量和接口接受度。
- 完成数值收敛、跨求解器比较、GUI可检查性、CAD同步、运行manifest和失败证据归档。
- 正式化不是文件改名，而是将通过门禁的设计身份、模型、CAD、结果和来源运行共同发布。

## 设计范围

长期设计范围包括但不限于：

- TOF质量分析器：正交加速TOF、不同反射次数的TOF和MR-TOF等独立设计线；
- RF多极杆离子光学：四极杆、六极杆、八极杆及其质量分析、碰撞、冷却、导引和聚焦功能；
- 离子与电子源：电子轰击离子源、Wehnelt电子枪及其他可参数化源；
- 接口与集成：部件间坐标、电位、孔径、压力、相空间、时间和机械接口匹配；
- 最终仪器：由已正式化部件和冻结接口组成的可验证系统设计。

设计族只表达共享物理和参数化方法，不自动共享同一正式资产。可以独立保持baseline、验收状态和正式
COMSOL/SIMION/CAD的硬件设计，是独立项目；同一硬件的运行工况和参数扫描则属于mode与run。具体
项目边界以仓库README为准。

## 正式交付定义

一次完整正式交付按适用范围至少包括：

- 机器可读的需求、baseline、resolved参数、分析和接口合同；
- 可重建且GUI可检查的COMSOL、SIMION和必要CAD资产；
- 输入、软件版本、随机种子、输出与哈希完整的运行记录；
- 性能、收敛、鲁棒性、跨求解器和接口验证结果；
- 未满足指标、适用边界和已知非阻塞限制；
- 正式资产manifest以及产生它的来源run。

并非每个项目都必须使用全部软件；正式交付要求的是对声明能力充分且可复验的证据，而不是形式上的
工具数量。

## 边界与非承诺

- 愿景中的数值仅可作为需求示例。类似“500 Da、单电荷、分辨率30000”的表述在获批并进入项目
  机器合同前，不是当前基线或完成声明。
- 平台不以自然语言直接修改正式模型；所有变更必须先形成可审阅合同和候选运行。
- 平台不保证任意目标在给定物理、尺寸、电压、成本和制造约束下可实现；不可行性结论也是正式结果。
- 自动优化不替代物理审计。代理模型、启发式算法和单求解器最优点必须接受独立验证。
- 本文不承诺日期、当前完成度或具体实现模块；这些内容分别属于Roadmap、项目状态和未来架构文档。
