# 科学绘图与图形证据规范

本文是仓库内科学绘图、仿真诊断图、跨求解器比较图、报告图和发表图的唯一跨项目规范。仓库架构、
artifact生命周期和文件角色仍由根[`README.md`](../README.md)定义；编码与机器合同遵循
[`DEVELOPMENT_STANDARDS.md`](DEVELOPMENT_STANDARDS.md)；指标、统计和跨求解器验证语义遵循
[`VALIDATION_METHODS.md`](VALIDATION_METHODS.md)。Agent权限和面向用户的交接方式只由
[`AGENTS.md`](../AGENTS.md)定义。

目标期刊、客户或演示模板可以覆盖尺寸、字体和输出格式，但不能降低数据真实性、变换透明度、
可比性、可访问性、可复现性和来源追溯要求。

## 适用范围与图形等级

本规范适用于Python、MATLAB、COMSOL、SIMION及以后实际引入的工具所生成的定量图形。图形等级只控制
图形证据深度，不等于项目的Static、Candidate或Formal生命周期：

|等级|用途|最低要求|
|---|---|---|
|探索图|当前分析会话中的临时观察，不被文档、结论或他人引用|不修改原始数据；不得冒充证据；任务结束时按scratch规则处理|
|运行诊断图|单次run内定位求解、接口、损失或数值问题|进入run manifest；量名、单位、样本数和状态语义明确；任何变换可追溯|
|证据图|支持项目结论、参数选择、求解器比较或验证门禁|稳定语义文件名；冻结来源和指标定义；比较条件一致；完成自动检查和实际尺寸人工复核|
|报告/发表图|面向报告、演示、客户或期刊的可交付图形|满足证据图要求，并按目标媒介验证尺寸、字体、格式、caption和导出质量|

临时图一旦被PROJECT、报告、提交说明、演示或正式讨论引用，就升级为运行诊断图或证据图并满足对应
要求。COMSOL或SIMION原生GUI图可以承担模型内诊断和GUI对等证据，但不能绕过相同的科学诚信要求。

本规范对新建或实质修改的图形、绘图数据合同和导出入口立即生效。既有immutable run不为统一外观而
原地重画；旧源码未被相关任务触及时不批量格式化。既有图一旦被重新发布、用于新的验证声明，或其
数据语义、坐标、颜色、版式和导出职责被修改，就应在同一任务迁移到本规范并生成新的可追溯证据。

## 核心原则

1. 科学含义优先于装饰效果；每张图和每个面板只有明确的证据目的。
2. 原始数据、指标计算、绘图数据准备、渲染和导出是可区分的职责。
3. 过滤、排除、裁剪、插值、平滑、归一化、分箱、拟合、坐标变换和降采样不得隐藏。
4. 被直接比较的案例必须统一样本定义、单位、坐标、时间原点、范围、分箱、归一化和指标口径。
5. 颜色不能是唯一的信息载体；最终显示尺寸下的文字、线条、标记和图例必须可读。
6. 证据图由代码和冻结数据生成，不允许手工修点、重画、拼接或无记录地编辑定量内容。
7. 图必须能追溯到来源run、输入、数据、指标定义、生成代码和适用的样式版本。
8. 降采样和栅格化只服务于显示性能，不能改变指标或隐藏与结论有关的结构。
9. 图不能替代机器可读结果、summary、manifest或PASS/FAIL判据。
10. 生成式图像不得被用作、混入或替换定量仿真与实验图形证据。

## 责任边界与依赖方向

推荐依赖方向是：

`求解器原始输出 → 项目canonical结果 → 指标分析 → 绘图数据准备 → 渲染 → 导出`

- 求解器实现负责原生模型、原始输出和GUI可检查的诊断节点，不维护跨求解器发表图的第二套指标。
- 求解器无关指标、统计和坐标转换在项目分析层或已验证公共层计算，不藏在低层绘图调用中。
- 绘图数据准备验证列、单位、状态、有限值、分类顺序和变换，渲染只消费明确的数据。
- 小型项目允许把准备、渲染和导出保存在同一模块，但应使用职责清楚的函数，渲染函数不得静默重算
  FWHM、传输率、置信区间或其他结论指标。
- 供应商私有格式应先转换为项目canonical结果。一次性排错图可直接读取私有格式，但必须标明诊断范围，
  不能成为长期正式分析入口。
- notebook可以调用正式绘图函数，但不能成为证据图唯一且不可复现的实现。
- 不为本规范预建统一`src/plotting`、figure registry或空测试树。共享样式或绘图代码只有在至少两个项目
  实际采用并验证后，才按根README提升到`common/`。

## 命名与API

函数动词沿用开发标准，并使职责可从名称判断：

- `prepare_*`：验证、过滤、聚合或整形成明确绘图数据；
- `plot_*`：在给定或新建Axes上渲染，不保存、不显示、不启动求解器；
- `add_*`：增加单一注释、参考线、几何轮廓或局部视图；
- `format_*`：只改变轴与版式，不改变数据；
- `export_*`或`save_*`：持久化完成的图形及必要元数据；
- `build_*_figure`：由可测试面板组成一张多面板图；
- `validate_*`：失败关闭地校验图形或绘图数据合同。

禁止`make_plot`、`plot_data`、`helpers.py`、`final_figure.py`等职责不明或以生命周期代替语义的名称。
artifact内图形文件继续使用根README规定的稳定角色名，例如`mass-spectrum__peak-overlay.png`；不得用
`final`、`latest`、`new`、`v2`或人工日期管理版本。

新建或实质修改的Python证据图优先使用显式Figure/Axes接口。渲染函数返回Figure和Axes，不调用
`show()`、不保存文件、不依赖当前Axes或notebook全局状态；多面板图优先由可独立测试的面板函数组成。

## 图形数据合同与追溯

运行诊断图和证据图必须能确定：

- 来源run ID、输入合同、结果路径和相应哈希；
- 字段名称、单位、坐标系、参考frame、时间原点和必要变换；
- 总样本数、有效数、命中/到达数、损失数、无效数和百分比的实际分母；
- 过滤、排除、裁剪、归一化、分箱、拟合、平滑、插值和不确定度方法；
- bootstrap、jitter、随机抽样或确定性子集所用的种子和选择规则；
- 类别顺序、颜色/线型/标记语义、轴范围和适用样式profile。

优先由现有`run_config.json + summary.json + run_manifest.json`共同提供以上身份，不为相同数据维护第二份
manifest。以下任一情况发生时，应增加具名figure manifest或等价的报告图合同：

- 一张图组合多个run或多个项目；
- 图形变换、caption、样式或来源映射无法由run三件套完整恢复；
- 图被独立交付，脱离父run后仍需验证身份；
- 目标期刊、客户或报告流程要求逐图资产清单。

figure manifest至少记录图形目的、来源run和哈希、Git身份、工作树状态、尺寸、格式、DPI、单位、
坐标、过滤、分箱、拟合、不确定度和随机选择。可审阅的小中型可重建数据优先CSV；只有性能测量证明
必要并建立版本化Schema后才使用Parquet或HDF5。不得复制大型原始数据制造第二份权威；可以引用已由
run manifest冻结的数据及其哈希。

## 科学诚信与统计表达

- 原始数据保持不变；所有显示变换在代码和元数据中显式实现。
- 禁止为获得期望视觉结果手工删除粒子、点、轨迹、运行或尾部数据。
- 离群值政策应在观察目标结果前定义，并与指标合同一致；过滤图应报告保留数和排除原因。
- 平滑不能作为噪声数据的唯一表示；与结论有关时同时显示原始点、固定分箱或不确定度。
- 拟合曲线必须与观测数据可区分；拟合承担结论时同时给出残差或适当的拟合质量。
- 插值不得暗示采样域外存在数据；外推区必须明确标出。
- 均值、中位数、拟合、模型预测和单次运行不得使用含糊相同的线/点语义。
- 误差条和阴影必须说明是标准差、标准误、置信区间、预测区间、四分位距还是运行范围。
- 传输率等二项量只有在报告不确定度时才选择并声明适当区间，不能用未定义的误差条。
- 粒子数不同的分布比较不得默认使用原始直方图计数；除非样本数差本身就是信息，否则使用概率、
  密度或明确归一化计数，并显示实际N。
- 柱形图和面积图的定量轴原则上从零开始；散点或折线可以截取范围，但不得夸大结论或隐藏观测。
- 断轴强烈不推荐；确有必要时必须明显标记并在caption或元数据中解释。
- 双y轴原则上不用；优先对齐面板或规范化变量。确需双轴时必须使两个独立尺度显式，不能暗示伪相关。
- 放大图必须在主图中标出被放大的区域；轴范围不得无记录地隐藏数据。

## 图形类型选择

选择能保留科学结构的最简单图形：

|问题|优先形式|避免|
|---|---|---|
|连续变量关系|散点；有序x的折线；必要时残差面板|连接无序样本|
|粒子结果分布|固定分箱直方图、ECDF、点/rug|只显示平滑密度|
|离散类别比较|点区间图、点图；真实总量才使用柱形图|装饰性饼图/环图|
|扫描及不确定度|实际采样点加区间；必要时连线|只显示最佳点或未标注阴影|
|大规模二维密度|hexbin、二维直方图、栅格化散点|数百万个不透明标记|
|标量场|image、pcolormesh或等值线加定量colorbar|无定量尺度的彩虹图|
|矢量场|一致抽稀的箭头/流线加幅值背景|遮挡结构的密集箭头|
|求解器一致性|parity、差值/残差、适用时Bland–Altman式视图|只有两条重叠曲线|
|性能与缩放|原始重复、稳健汇总、拟合及残差|只有拟合线|
|收敛|误差对网格/步长/容差/N及实际采样点|凭一次比较声明收敛|

三维轴只用于真正需要三维几何或轨迹语境的图。定量比较优先采用更清楚的二维投影、切片或残差。

## 坐标轴、单位、比例和刻度

- 每个定量轴、colorbar和可解释的图例都必须写明量名与单位，例如`Flight time (µs)`、`E_z (V/m)`。
- 单位标签默认使用括号；数值注释的数值与单位间保留空格。无量纲量使用已定义符号或明确标注。
- 坐标含义可能混淆时写明平面、分量和frame，例如`x–z plane`或`local E_z`。
- 默认使用线性轴。log、symlog或其他非线性轴必须具有物理理由、合法数据和可读刻度，并记录原因。
- 刻度精度与输入、数值或统计精度匹配，不显示无意义小数。
- 用于直接比较的面板统一范围和主要刻度；独立自动缩放不能用于定量比较。
- 网格线保持克制，只在帮助读取数值时使用。
- 几何、场切片和轨迹投影使用相等或物理上有意义的比例；若为展示改变比例，必须显式披露。

## 颜色、色图与可访问性

- 类别同时使用颜色和线型、标记、直接标签或面板位置中的至少一种；禁止只用红/绿区分。
- 报告和发表图默认白色或近白背景；诊断或演示可采用深色主题，但文字和关键线条必须有足够对比度。
- 图形应在灰度打印时仍可解释；不使用彩色文字代替清楚的图例样本。
- 默认定性颜色顺序为蓝`#0072B2`、橙`#E69F00`、绿`#009E73`、朱红`#D55E00`、
  天蓝`#56B4E9`、紫`#CC79A7`、黄`#F0E442`和黑`#000000`。白底上的细线、小标记或文字不使用黄色。
- 同一报告或同一受控研究中，类别顺序和颜色/线型映射保持稳定。
- 有序量使用感知均匀的顺序色图，如`viridis`、`cividis`、`magma`或`inferno`。
- 有意义中心两侧的差值使用显式中心规范化的发散色图；相位使用周期色图；类别不用连续色图。
- 禁止`jet`、普通rainbow和以色相循环表达有序量。
- 定量比较的场图共用色标范围、规范化和等值线；局部对比确需独立色标时必须明确标注，不能宣称
  直接幅值一致。
- 饱和、裁剪和无效区必须在colorbar、图形或caption中可见。

## 字体、尺寸与多面板布局

证据图先确定最终媒介和物理尺寸，再调整内容。以下是无目标模板时的默认profile：

|profile|宽度|用途|
|---|---:|---|
|`publication_single`|89 mm|期刊单栏|
|`publication_double`|183 mm|期刊双栏|
|`report`|160 mm|技术报告|
|`presentation_16_9`|300 mm|演示或大屏|

报告/发表图在最终尺寸下默认使用可用且许可兼容的无衬线字体：刻度和图例约8 pt，轴标签约9 pt，
需要时的面板标题约10 pt，面板标签使用9–10 pt加粗小写字母；关键文字原则上不低于7 pt。线宽通常
0.8–1.5 pt且不低于0.5 pt，标记通常3.5–6 pt。目标媒介可以覆盖这些默认值。

- 每张图只表达一个主要消息，每个面板职责明确；按从左到右、从上到下排列。
- 面板标签位置稳定；共享轴标签、图例和colorbar只在不产生歧义时合并。
- 图例不得遮挡数据；顺序与视觉、扫描或物理参数顺序一致。
- 避免阴影、渐变、装饰性3D、粗重网格、重复标题、图标和无意义留白。
- caption承载方法、条件和解释，图内只保留简洁量名、图例和必要注释。
- 使用受控布局，不用大量任意像素偏移修补；导出前检查标签、图例、注释和colorbar没有裁剪或重叠。

## 质谱仿真专用要求

### TOF、峰形和分辨率

- 记录总粒子、到达、拒绝和实际绘图分母；跨求解器直方图使用相同分箱边界和显示窗口。
- 明确纵轴是计数、概率还是概率密度。
- FWHM、质心、峰位和分辨率只使用仓库指标合同；拟合峰不能替代观测分布。
- 分析窗口影响峰宽或尾部时保存并显示窗口定义；偏斜、肩峰或多峰不能用单一高斯宽度冒充完整描述。

### 传输、参数扫描和损失

- 定义传输分子、分母、终止状态和无效粒子处理；连续参数按数值排序。
- 显示实际采样点；连线只表示扫描顺序或显式插值，不表示额外模拟。
- 重复运行显示变异，不只展示最好结果；密集连续扫描不用柱形图。
- 损失图保持粒子ID、终止实体、终止原因和坐标，不能把所有未命中合并成无位置的单一数字。

### 跨求解器比较

- 声称粒子级一致时使用共同粒子ID和等价初始条件，并统一单位、frame、时间原点、检测面和状态映射。
- 直接比较面板统一范围，并至少提供差值、相对差、parity或残差之一。
- 分母接近零的相对误差必须有明确政策，不能输出不稳定百分比。
- solver名称、版本、场来源和粒子数进入manifest或caption；共同场积分器测试不得标成独立场闭合。

### 轨迹和场图

- 大轨迹集按粒子ID规则或固定种子确定性抽样；禁止手挑有利轨迹，并记录原始数、显示数和选择方法。
- 电极、孔径、屏蔽和检测面使用与轨迹相同的坐标变换；保持物理比例且避免几何遮挡轨迹。
- 用颜色表达时间、能量、质量或状态时提供colorbar或明确图例。
- 场图记录场来源、网格/采样、插值、切片平面、frame和分量；不在有效域外插值或隐藏无效区。
- 比较场图使用共同色标和等值线；矢量箭头一致抽稀且不能替代幅值色标。

### 性能与收敛

- 性能图展示原始重复或其分布，区分总墙钟时间和求解阶段时间，并记录机器、硬件、内存、版本、
  重复次数和冷/热启动政策。
- 跨机器性能比较只有在执行条件可控时成立；外推区必须单独标出。
- 收敛图写明被扫描量、参考解、误差指标和实际采样级；拟合阶数必须处于有依据的渐近区。
- 数值收敛与有限粒子/Monte Carlo统计收敛分开表达。

## Python与供应商绘图实现

Python证据图默认使用仓库锁定环境中的Matplotlib，除非项目明确采用另一工具：

- 自动化和批处理使用非交互后端；库代码不调用`show()`，导出后关闭Figure。
- 使用显式Figure/Axes，不依赖跨函数的当前Axes状态，不在import时修改全局`rcParams`、生成图或写文件。
- 样式通过上下文或显式配置应用，不污染其他测试和图形。
- 直方图、区间、拟合、残差和抽样在渲染前计算并测试；不对可变DataFrame就地变换后直接作图。
- 大数据使用LineCollection、PathCollection、pcolormesh或等价批量artist，避免逐粒子创建大量对象。

MATLAB/COMSOL/SIMION可以保存原生诊断图和GUI节点；影响正式模型检查的PlotGroup、dataset、派生值和
导出继续遵循[`COMSOL_API.md`](COMSOL_API.md)。求解器原生图不重新定义跨软件指标；跨求解器和最终
报告图优先从canonical数据生成。软件能力不足时明确记录未实现项，不伪造视觉对等。

## 导出、位置、caption和figure manifest

- 图形进入所属run的`results/`或经门禁提升后的`formal/results/`；是否增加语义子目录由该run的输出
  规模决定，不建立根级`runs/`或第二棵artifact树。
- 线条、文字和矢量几何优先PDF或SVG，并按工作流提供PNG预览。普通运行诊断PNG不低于约180 dpi；
  报告和发表用彩色/灰度栅格通常不低于300 dpi；无法使用矢量的线稿通常不低于600 dpi。
- JPEG不用于科学图、线稿或文字密集图。白色不透明背景是默认；透明背景必须针对目标背景复核。
- 密集场图或散点可以在PDF/SVG内只栅格化数据artist，保留轴、文字和注释为矢量。
- 每次导出显式指定格式、尺寸和DPI；扩展名不能代替格式合同。`bbox_inches='tight'`改变名义尺寸，
  只有在复核最终尺寸后使用。
- 可引用run不可原地覆盖图形证据；重新计算产生新run，或按README定义的生命周期晋升和取代。
- 新生命周期实现优先写临时文件、验证能打开、尺寸和必要内容后再原子发布。

报告/发表图的caption或caption draft应说明：图的目的、案例和求解器、关键条件、样本数和分母、
线/点/阴影/误差条语义、变换与排除、指标和不确定度定义，以及各面板内容。不要把完整caption复制成
图内大标题。

## 大数据和渲染性能

1. 数据解析、聚合和指标只执行一次，多种导出格式复用同一准备数据。
2. 先在完整数据上计算指标，再进行显示降采样；除非指标合同明确要求，否则不得先抽样再算指标。
3. 降采样记录原始数、显示数、选择规则和种子；完整分析数据仍由run保存或引用。
4. 大表按块读取且只加载需要列；缓存键包含源数据、代码、配置和指标定义的内容身份。
5. 批处理及时关闭Figure，不通过全局注册或闭包长期保留大数组。
6. 只有建立正确性baseline并测量热点后才优化渲染，不因源码长或肉眼感觉慢而改变数据语义。

## 自动测试与人工复核

绘图测试优先覆盖数据准备和结构，不依赖脆弱的逐像素相等：

- 单元测试分箱、归一化、排序、单位、类别顺序、不确定度和确定性抽样；
- headless小fixture执行渲染smoke test；
- 检查必需的轴标签、单位、图例、colorbar、面板标签、共同范围和固定分箱；
- 空数据、NaN/Inf、缺列、非法单位和未知类别失败关闭；
- 随机bootstrap、jitter和显示抽样在测试中固定种子；
- image regression只用于少数稳定、高价值图，并区分字体/后端造成的无害像素差异；
- 绘图单元测试不启动昂贵求解器；端到端图形测试显式消费小fixture或已验证run。

证据图和报告/发表图在最终显示尺寸下还要人工检查：结论是否清楚、量名和单位是否正确、样本与变换
是否披露、比较尺度是否一致、颜色是否可区分且不唯一、几何比例是否正确、观测是否仍可见、是否存在
裁剪/重叠、文件能否在目标viewer打开，以及来源数据和代码能否重建可见marks。自动测试、人工图审和
真实求解器验证是三个不同证据层级。

## Agent工作流

创建或实质修改证据图前，Agent应：

1. 读取本文、相关指标合同和现有样式/相邻图形；
2. 确定探索、运行诊断、证据或报告/发表等级及目标受众；
3. 定位canonical数据、来源manifest、frame和指标定义；
4. 搜索现有图形角色、准备函数和类别语义，避免第二套实现；
5. 明确本次只改变视觉表达，还是改变数据语义、过滤、指标或比较合同。

修改期间不得为了更好看而改变轴范围、颜色、归一化、平滑、排除或样本；确有科学理由的改变作为
显式合同变更验证。完成后运行适用的数据测试和渲染smoke test，生成代表性预览，检查尺寸、格式、
可读性和裁剪，并在交接中报告来源数据、实质语义变化、验证层级和产物；普通实现细节仍按AGENTS和
用户偏好保持紧凑。

未检查最终尺寸和目标媒介要求时，不得称图为`publication-ready`。

## 完成条件

运行诊断图至少证明：来源run可追溯，量名/单位/状态/样本数正确，变换显式，输出被run manifest冻结。

证据图还必须证明：目的和角色稳定，比较合同一致，图形类型与尺度适当，颜色可访问，必要数据可重建，
自动测试通过且完成实际尺寸人工复核。

报告/发表图还必须证明：目标尺寸、字体、线宽、格式、DPI、caption、viewer和媒介要求已经检查；任何
未验证的期刊专属要求或不可用字体明确列为待办。图形通过本规范不代表底层物理、求解器、Candidate或
Formal已经通过。
