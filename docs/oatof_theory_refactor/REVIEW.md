# oa-TOF 理论重写审查记录

## 结论

本目录原始投稿包的三份 Markdown 和三份 Python 均已完整审查，`SHA256SUMS.txt`覆盖的原始文件
哈希验证通过，三份脚本自检通过。原稿保持不变；经修正和集成的活跃版本位于
`projects/oa_tof/docs/theory/`与`projects/oa_tof/analysis/`。

重写版本比旧 DOCX 有更高的长期价值，原因是它明确拆分了：

1. 加速器局部一阶空间聚焦；
2. 反射镜局部一、二阶能量聚焦；
3. 保留加速器时间项的整机纵向耦合。

它还显式定义参考面、能量每电荷、包络、第二级深度、FWHM 禁止性结论和 L0--Formal 升级边界，
适合成为机器合同和回归测试的理论源。

## 已确认并在活跃版本修正的问题

- 反射镜三阶端点时间公式在使用 mm 时漏写 `10^-3` SI 换算因子；Python 实现原本正确。
- R. Stein (1974) 的页码应为 `205--218`，不是 `355--364`。
- 加速器合同把非整数 `ring_count` 静默 `int()` 截断；现改为拒绝。
- 耦合合同把非整数 `sample_count` 静默截断，CSV 索引也写成浮点；现改为严格整数。
- 加速器候选合同要求 canonical 坐标，但新旧实现都没有输出；活跃实现已补齐并加入回归。
- 耦合根求解曾使用带量纲的固定绝对残差提前退出；现改为区间收敛加最终尺度化残差检查。
- 旧反射镜 API 使用 m 和字典返回值；集成后统一为 mm 和数据类，现有候选编译调用已同步。

## 独立验证

- 加速器焦面导数、反射镜一/二阶导数和整机耦合一/二阶导数均以独立五点有限差分复核；
- 在确定性随机物理域中检查了闭式解、导数和耦合根，不依赖脚本自己的 self-test；
- 当前 `2000 V / L=1200 mm / stage1=120 mm` baseline 可复现局部解
  `U_R1=1600 V`、`F_2=9.213106741667369 V/mm`；
- 包含现有加速器二阶曲率后，诊断耦合解为
  `U_R1≈1628.800063 V`、`F_2≈9.384721788 V/mm`，与 Formal baseline 不同；
- 按完整 `1920--2080 V` 空间能量包络再施加现有 100% 深度裕量时，需要第二级深度约
  `96.156273 mm`，当前 `86.8328 mm` 不满足该重新定义的严格裕量合同。因此它是开放的裕量语义/
  候选设计问题，不是现有 Formal 结果的自动否定，也不能静默改写 baseline。

机器可复算诊断合同为
`projects/oa_tof/config/candidates/oatof_longitudinal_coupled_reference.json`。
