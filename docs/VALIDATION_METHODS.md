# 跨求解器验证方法

本文件只保存换一个项目仍成立的验证方法。项目参数、某次运行数值和转正结论写入项目
`docs/PROJECT.md`。

## 输入对齐门禁

比较两个求解器之前，先用机器契约逐项核对：几何尺寸与坐标、器件变换、电势、粒子质量与
电荷、释放位置、能量和方向分布、有效检测面、命中条件及单位。优先让两个求解器读取同一份
固定粒子表；否则随机误差和输入差异无法分离。

几何一致不是“看起来相同”。长期门禁应从契约读取关键尺寸，并在各软件源文件或运行时模型中
验证派生关系。迁移一个器件时，其电极、接地屏蔽、绝缘间隙、检测面和选择集必须参数化联动。

## 分辨率与峰形

质量分辨率统一为

`R = m / FWHM_m`。

窄峰并满足飞行时间与质量平方根关系时，时间域等价式为

`R = T / (2*FWHM_t)`。

`FWHM = 2.3548*sigma` 只对近似高斯峰成立。偏斜、肩峰、双峰或截断峰应直接在强度曲线上求
半高交点，并同时保存强度-时间谱、强度-质量谱和检测器落点分布。直方图区间必须固定或报告，
避免不同自动分箱造成表观 FWHM 差异。

## 统计误差

固定输入和网格后，用多个随机种子或自助重采样估计命中率、均值和 FWHM 的置信区间。只有两个
求解器的差值落入差值置信区间，才能称为统计上一致；单看百分比大小不能判断“偶然误差”。峰形
非高斯时优先对直接 FWHM 做重采样，不用 sigma 误差替代。

## 网格误差

每个求解器独立做网格收敛。一次只改变目标区域的网格，保持几何和粒子完全不变，比较命中率、
平均 TOF、直接 FWHM、分辨率及峰形。SIMION 各向异性 PA 和 COMSOL 非结构网格的单元定义不同，
不能用相同的毫米数直接宣称误差相同。

先用计算较快的求解器定位敏感区域和无效加密，再在较慢求解器中选择最少的粗/中/细组合闭合。
若指标变化已小于统计不确定度和工程容差，应停止无目的加密。

## 最小交付

一次跨求解器比较至少保存：输入契约版本、软件/模型版本、粒子表或种子、网格、命中数、平均
TOF、直接时间与质量 FWHM、分辨率、峰形图、落点图和验证脚本 PASS/FAIL。最终跨软件结论只写
入 PROJECT，各软件独立实现细节留在各自文档。
