# COMSOL/MATLAB 通用排错

本文件只回答“COMSOL实现为什么失败或表现异常，以及怎样定位到具体节点、选择、Study、Solver、
会话或进程”。准确的Feature类型、属性和调用写法见[`COMSOL_API.md`](COMSOL_API.md)；网格收敛、
统计不确定度、解析场干预和跨求解器闭合见[`VALIDATION_METHODS.md`](VALIDATION_METHODS.md)。
项目参数、某次运行数值和历史结论留在项目文档。

本文结论以Windows、COMSOL 6.4、MATLAB R2025b和本仓库统一启动器为当前适用环境。与渲染器、
许可证或内部类型名有关的经验升级版本后必须重新做最小测试，不能当作永久产品事实。

## 快速分诊

先确定失败发生在哪一层，再决定检查对象：

|现象|第一检查点|不要先做|
|---|---|---|
|Feature或属性报Unknown|GUI Record Code、`properties()`、最小模型|在正式大脚本中连续猜名字|
|几何成功但边界条件异常|命名选择、实体数量、选择窗口|先全局加密网格|
|Compute成功但场为零|Study激活状态、初值/解引用、数据集|先修改电极电压|
|粒子不动或直线飞行|Force selection、所用场数据集、质量/电荷|仅凭轨迹图判断渲染坏了|
|求解完成但提取失败|解是否已保存、数据集类型、返回数组大小|自动重算整个模型|
|图片导出卡死|数据是否存在、批处理渲染限制|反复强制软件渲染|
|保存报文件锁|目标文件、`ComsolUI.exe`和服务端会话|未经确认强杀GUI|
|连接反复失败|启动阶段、任务报告和服务端日志|把所有失败无限重试|

每轮只改变一个假设，并记录输入、失败阶段和结果。若问题已经从“实现是否正确”转为“两个正确
模型是否等价”，停止在本文件继续试错，转入validation流程。

## 属性和Feature发现

不确定属性或合法取值时，优先顺序是：

1. 当前版本GUI中手工建立最小节点并使用Record Code；
2. 在scratch模型中读取`feature.properties()`，逐项用`getString()`或相应getter检查默认值；
3. 查本机当前版本Programming Reference、Module User’s Guide和Application Library示例；
4. 仍不明确时，在最小模型中故意传入一个无效值，利用异常给出的合法取值缩小范围。

错误值探测只用于一次性最小实验，不能成为正式脚本的运行时发现机制。候选Feature Type需要批量
试探时，也应在空白scratch模型中完成，不在已经包含复杂几何和求解历史的正式模型中循环创建。

大块功能先拆成最小测试：创建节点、设置selection、建立最小Study、Compute、保存并重开。最小
测试通过后再集成到正式生产脚本，避免每次错误都重跑前面已经验证的长流程。

## GUI模型树与持久化

脚本执行成功不等于模型正确。正式模型必须在保存并重新打开MPH后检查：

- 几何、材料、选择、物理条件、网格、Study和Solver节点存在且可编辑；
- Study的GUI Compute使用预期物理激活状态和Solver；
- 数据集、派生值、绘图组和导出引用的是已保存解；
- 关键参数不是只留在MATLAB变量或Server会话中。

若`model.sol(...).runAll`成功而GUI Compute不同，优先检查Solver是否attach到Study、Study Step激活
状态和初值引用，不用后处理结果相近来掩盖模型树不等价。

## 几何和选择集

### 几何移动后的选择同步

基于绝对坐标的`Box`选择不会随目标几何自动移动。器件平移、缩放或参数化后，检查所有Box窗口、
释放区域、局部网格选择和检测边界是否联动。基于拓扑的`Adjacent`通常更稳健，但仍须验证实体数。

### 尺度变化与选择窗口

固定宽度的选择窗口在间距缩小时可能框入相邻边界，使不同物理条件争用同一实体且不一定报错；
尺寸增大时又可能漏选。每次几何尺度变化后打印关键选择的`entities()`数量，并抽查实体坐标、法向
和相邻域。数量异常优先怀疑窗口重叠、拓扑改变或几何容差，但不能不经检查就断定唯一根因。

两个条件选到同一边界是否影响结果，要同时检查电位/物理条件是否相同以及关键场量是否变化。
“电位相同”只能说明眼前可能无数值影响，不能让错误选择继续充当正式模型契约。

### 材料和域选择

几何实体存在不代表材料、Force或物理接口自动覆盖该域。材料、场接口、粒子力和局部网格都要
显式selection并验空。空selection可能不报错，只产生零场、零力或默认材料。

## 场和边界条件

场的符号、数量级或对称性异常时，按以下顺序检查：

1. 电极/端子的命名选择和实体数量；
2. 电势、接地、材料单位和参数实际值；
3. 导体是否因容差意外接触、合并或形成错误封闭域；
4. 物理接口的域selection与Study激活状态；
5. 所查询数据集是否对应预期解；
6. 最后才检查网格是否不足。

怀疑屏蔽或边缘场时，先沿轴线和代表离轴点提取势与场，不只看颜色图。接地开孔平板不等于理想
屏蔽；是否需要套筒、多级电极或改变孔径属于项目设计结论，应在项目中通过受控候选验证，不能把
某一尺寸经验写成通用COMSOL修复。

## Study、Solver和解引用

时间相关粒子Study中仅把静电接口设为inactive，不会自动保证复用稳态场。未求解变量可能回退到
零初值，粒子仍会以初速运动且没有明显报错。排查时在同一物理坐标分别查询稳态数据集和粒子Study
数据集的场值；若后者处处为零，检查Solver的`notsolmethod`、`notsol`、Study attach和变量来源。

自定义Solver序列创建后必须显式关联Study。检查GUI Compute，而不只检查脚本直接运行Solver。
修改Study Step、物理激活或初值引用后，删除或隔离旧Solution，避免后处理静默读取上一次成功解。

Compute成功但派生值或导出失败时，先确认解已存在并持久化，再用干净任务做轻量提取。大粒子数组
提取失败不应自动触发昂贵场和轨迹重算。

## 粒子追踪

### 粒子不动或力为零

按顺序检查：释放特征是否产生粒子、质量/电荷和初速度是否有单位、Force selection是否覆盖真空
粒子域、Force引用的场是否来自正确解、Wall是否提前终止。配置磁力后粒子速度大小守恒是正常的，
不能用“磁力不做功”证明轨迹正确；仍要检查曲率、方向和回旋半径。

### 数据存在但轨迹图为空

先用`mphparticle`或结果信息确认粒子位置、时间和线段数确实变化，再判断为渲染问题。若坐标没有
变化，回到Study/Force排查；若数据真实存在而PNG为空，保存含原生Particle数据集和PlotGroup的MPH，
在Desktop查看，或用MATLAB绘图引擎从粒子数据生成诊断图。

### 终止和NaN

NaN、缺失末态或短轨迹要按Wall、离域、超时和求解失败分别统计，不能删除后宣称全部命中。跨软件
命中与终止语义的统一属于validation，不在COMSOL后处理中临时改分类迎合另一求解器。

## 网格故障

网格构建失败或单元数异常时，先检查几何是否形成极薄域、短边、重合面或不合理小间隙，再检查
局部Size selection是否正确。参数化几何的局部精细区应由同一参数派生；固定半径会在尺寸增大时
覆盖不足，在尺寸减小时导致单元数暴涨。

`hmax`是最大尺寸约束，不代表所有单元固定等大。网格成功后仍要检查单元数、最小质量、目标域
覆盖和空网格。粗/中/细收敛、跨求解器网格误差和何时停止加密统一按validation执行。

## 卡死、崩溃和结果保存

### 批处理粒子轨迹图片

在当前Windows/COMSOL 6.4无GUI批处理中，`ParticleTrajectories`图片导出可能卡死，而普通场图
导出正常。当前环境曾观察到强制`-3drend sw -graphics`可让单次导出返回，但随后可能触发
`csgraphics_sw.dll`访问违例并使服务端崩溃，因此不得把该开关作为长期服务端的默认修复。升级
COMSOL或显卡环境后需要重新最小验证，不能照搬这一环境结论。

有风险的可视化放在数值求解和`model.save()`之后，并用try/catch隔离。最有价值的解先落盘，渲染
失败不能抹掉已经完成的计算。

### 强杀后的服务端

客户端被强制终止后，服务端可能残留会话、锁或未释放资源。正式任务使用统一启动器创建干净任务，
不连接可疑旧端口继续工作。人工服务端若必须复用，先运行几秒内应返回的最小健康检查；失败则重启。

连续多次kill/restart后才出现网格或求解卡死，而全新启动总能成功，优先怀疑句柄、端口、Java内存
或内存映射未释放，不要继续无限自动重试同一业务脚本。

### 文件锁

重启`comsolmphserver.exe`后仍无法保存时，检查是否有`ComsolUI.exe`持有目标MPH。GUI进程可能是
用户主动打开且尚未保存的窗口；技术文档只说明锁来源，终止权限统一遵循`AGENTS.md`，不得把进程名
看起来可疑当作强杀授权。

## 故障分类与重试边界

|失败阶段|证据|处理|
|---|---|---|
|尚未进入任务脚本|没有任务报告，连接初始化失败|统一启动器最多自动重试一次|
|业务脚本已写FAIL|报告含稳定错误和失败步骤|不自动重算，修输入、节点或接口|
|Compute成功、提取失败|解或MPH已存在|保留解，用干净任务轻量提取|
|进程退出但无报告|退出码、服务端日志、文件时间戳|按启动失败审计，不凭进程消失判成功|
|模型保存成功但GUI Compute不同|模型树、Study/Solver引用不一致|修GUI对等，不用脚本结果替代|
|交付哈希不符|字节变化或文件被替换|按validation的文件身份规则判断|

项目脚本开始后出现确定性错误，不得由启动器自动重算；否则既浪费时间，也可能覆盖最有诊断价值的
第一次失败证据。

## 磁场、线圈和磁力

磁场模型按“接口可创建 → 线圈前置条件满足 → 场解量级正确 → 粒子真正受力”分层排查。合法接口、
Coil属性、Study类型和力特征名以`COMSOL_API.md`为准，不在本文件复制第二份名称表。

Numeric Coil错误常逐层暴露：端子/方向selection为空、材料缺少电导率、线圈几何尚未由所需Study
步骤求解。每修一层就在最小模型中重跑，不一次改多个节点。场解成功后用有限长或无限长螺线管
估算检查方向和量级；数量级错误优先查电流方向、匝数、单位、材料和selection。

粒子仍走直线时先查`MagneticForce`作用域和所引用磁场解，再用`r=mv/(|q|B)`、旋转方向和轨迹
完整性验证。计算域必须容纳预期圆弧，避免边界截断被误判为回旋半径错误。

## 排错证据与知识归档

可复用故障基线应保存：最小触发脚本、软件版本、失败阶段、关键异常、确认的根因、修复后PASS和
适用范围。API名称事实进入`COMSOL_API.md`；跨求解器或统计方法进入`VALIDATION_METHODS.md`；只在
一个项目出现的几何和物理结论进入项目COMSOL或PROJECT文档。不要把一次项目数值提升为根级规则。

进程终止、文件删除、覆盖和Git权限不在技术排错文档重复规定，统一遵循仓库`AGENTS.md`和根README。
