# MR-TOF SolidWorks 实施说明

## 当前资产边界

2026-07-21 接收的 CAD 是用户自绘、未经项目门禁验证的原始设计种子。文件整体冻结在
`artifacts/projects/mr_tof/archive/` 的迁入快照中，保留原文件名和同目录关系，避免人为破坏
SLDASM对SLDPRT的相对引用。该快照只读保留，不作为后续脚本或人工修改的工作目录。

## 首次审计门禁

开始参数化或几何修改前，应使用仓库规定的 SolidWorks 2022 工具链完成：

- 识别顶层装配及全部子装配，并记录实际解析到的零件路径；
- 检查缺失引用、外部引用、抑制件、轻化件、配置、单位和重建/保存警告；
- 建立 MR-TOF 本体与离子源、注入/引出、真空、检测和标准件的部件清单；
- 明确仪器轴、反射轴、注入面、引出面、检测面和共享坐标系；
- 从冻结快照复制到独立候选run后再修改，并在manifest中关联源快照哈希。

首次审计完成前，不把任何装配或零件晋升到`formal/cad/`，也不从现有几何反向猜测正式baseline。

当前项目状态统一返回[`PROJECT.md`](PROJECT.md)。
