# oa-TOF 当前项目状态

本文件是oa-TOF项目的人类可读权威源。机器参数以
[`../config/baseline.json`](../config/baseline.json)为准；COMSOL、SIMION和CAD文档只能记录
各自实现，不能在本文件之外另行定义项目参数。

## 当前统一定义

- 方案：正交加速飞行时间质谱，双级环栈反射镜，一级10环、二级5环。
- 标准离子：524 amu、+1电荷。
- 初始能量：均值5 eV、标准差0.4 eV，负能量重采样。
- 释放体：`1×1×1 mm³`，当前轴向范围`z=1...2 mm`。
- 分辨率：`R=m/FWHM_m`。窄峰时间域等价式为`R=T/(2*FWHM_t)`。
- `2.3548×sigma`只在高斯假设成立时作为FWHM代理；非高斯峰必须直接测量半高宽。
- SIMION常规统计N=5000；跨求解器严格比较优先使用同一固定ION粒子表。
- SIMION正式轨迹积分质量为8，由机器契约、同名Program加载段和IOB运行时门禁共同保证。
- SIMION稳定入口已冻结为0.05mm日常候选与0.025mm轴向收敛参考；实现资产与SHA-256见
  `config/simion_stable_entry.json`，它不替代`baseline.json`的物理契约。

### 参数派生硬规则

本项目严格执行：`物理输入 → 公式 → baseline工程参数 → COMSOL/SIMION/CAD`。机器权威源是
`config/baseline.json`；任何软件现有文件、网格坐标、格式化结果或GUI显示均无权反向修改它。

反射器长度由`U0=2000 eV`、总无场长度`L=1200 mm`、一级长度`d1=120 mm`和二级余量系数
`1.0`按baseline中记录的公式派生。原始计算为`d2=86.8328157299974 mm`、
`L_refl=206.832815729997 mm`；工程长度统一保留小数点后4位，因此baseline值只能是
`d2=86.8328 mm`、`L_refl=206.8328 mm`。COMSOL曾因`%g`序列化得到`206.833 mm`，这是实现
舍入错误，不是新权威值；该错误转正已撤销并归档。长期门禁必须重新计算上述公式，同时检查
COMSOL写入精度、SIMION实体/Program以及CAD均服从同一工程值。

## 当前几何状态

正式紧凑加速器保持轴向排布不变：孔半宽5 mm、环单边宽5 mm、带电电极到接地屏蔽
每侧5 mm、repeller后表面到接地后盖前表面5 mm、屏蔽壁厚4 mm；物理范围约
`38×38×30 mm³`。SIMION日常网格为`xy=0.25 mm、z=0.05 mm`，收敛参考为
`xy=0.25 mm、z=0.025 mm`。

外部屏蔽罩权威定义为COMSOL源码中的一体式封闭圆柱壳：内半径350 mm，侧壁及两端盖厚度
均为10 mm；近端真空面/外表面为`z=-30/-40 mm`，远端真空面/外表面按baseline派生为
`z=881.6628/891.6628 mm`。SIMION以无场区PA提供近端盖和连续侧壁，以反射区PA接续侧壁并
提供远端全截面端盖；Program只负责重叠PA区域的实例路由，不能改变上述几何。加速器出口栅网
是另一个独立的`30×30 mm`、`z=19.83 mm`理想透明栅网，不得误认成大屏蔽罩端盖。

2026-07-16已从修正后的源码重新生成524 amu、固定N=100、真实场、0.2 ns候选；100/100命中，
平均TOF为`71.9868802959 us`，直接质量FWHM为`0.017911007662 Da`，`R=29255.7521`。候选经
COMSOL参数、封闭屏蔽罩、出口栅网、Study/Solver及关键场值门禁后转为正式MPH。同一任务已用
SolidWorks 2022（revision 30.5.0）重建25个SLDPRT和25组件SLDASM，最大组件世界中心误差
`5.68e-13 mm`，保存错误/警告均为0。因此COMSOL与SolidWorks正式几何状态均为`synchronized`。
先前`206.833 mm`错误候选及旧CAD正式目录仅保留在archive追溯，不得恢复为权威入口。

SIMION检测器第4实例是半径40 mm、有效面`z=19.83 mm`的GUI可见数值终止层，不是机械
检测器实体。当前网格为`dx/dy/dz=0.5/0.5/0.01 mm`，终止层厚0.05 mm，前/背真空余量
0.2/0.05 mm。COMSOL/CAD机械检测器厚度仍按独立机械契约管理。

## 已闭合的SIMION结论

- 524 amu GUI Fly2、N=5000：5000/5000命中。
- 平均TOF约`71.9902651204 us`。
- 直接时间FWHM约`1.392926 ns`，对应`R=25841.38`。
- 直接质量FWHM约`0.02027719 amu`，对应`R=25841.84`。
- 时间域和质量域直接FWHM结果只差约0.0018%。
- 原生splat与`z=19.83 mm`参考面的最大逐粒子时间差约`4.0e-7 ns`。
- 峰为单个非高斯右肩峰；主要来源是初始z到TOF的二次映射，不是漏检、横向截断或绘图伪影。
- 2026-07-16原生GUI Recording已用Program On、Event=4、PA实例4和固定检测面完成5000粒子严格
  审计；Program Off配对诊断证明关闭Program会停用透明栅网跨越并破坏TOF映射，不能作为基线。

## COMSOL边界

当前正式COMSOL MPH仍是100 amu历史几何。按统一FWHM口径，其N=1000历史结果为1000/1000
命中、平均TOF`31.4478763926 us`、sigma`0.897802250264 ns`、FWHM`2.1141627354 ns`、
`R=7437.4304`。该结果不能标成524 amu，也不能代表紧凑加速器候选。

### 2026-07-15 固定粒子候选闭合（未转正）

同一份524 amu、N=100固定ION表已在SIMION quality=8与全三维COMSOL真实场候选中比较。COMSOL
`t=0`释放位置逐元素与ION表的最大误差为`7.1e-15 mm`，速度模长最大误差为`4.2e-4 m/s`，故两者
确实使用相同初始条件，不存在坐标列、单位或速度列导入错误。

两边均为100/100命中，平均TOF分别为`71.98684756 us`（COMSOL）和`71.99013526 us`（SIMION），
仅差`3.29 ns`。统一直接质量FWHM/KDE口径下，COMSOL为`0.01760645 Da, R=29761.82`，SIMION为
`0.01967453 Da, R=26633.42`；差约11.75%。时间步与SIMION trajectory quality不是可一一换算的
同一参数，因此不以强行调成同一R作为验收目标。

峰形比较图和原始到达时间保存在`artifacts/projects/oa_tof/results/cross_solver/`及
`artifacts/projects/oa_tof/runs/*fixed_particle_closure/2026-07-15/`。两边都是右偏主峰带右肩，
标准化KDE重叠为`0.869`、KS距离`0.140`，可判为主峰结构一致；但不是逐点闭合：COMSOL有约10%
高度的弱次肩、偏度更高，且TOF对初始z的相关性为`-0.691`，SIMION为`+0.373`。这说明剩余差异来自
同初始条件下的场/轨迹传递映射，而不是随机ION表导入。候选仍不得转正；后续应比较轴线场和初始z
到TOF映射，而非继续仅为压缩R差而减小COMSOL时间步。

### 2026-07-16求解器无关分析基线

项目已固定Python 3.11参考分析层，机器定义见`config/analysis_contract.json`，冻结数据身份和
回归值见`config/analysis_baselines.json`。正式直接FWHM统一为Gaussian KDE全局主峰两侧的线性
半高交点；带宽固定为`1.06*sigma*N^(-1/5)`，评价网格4001点。`2.3548*sigma`仍只作高斯代理。

四个外部基准的路径、行数、字节数和SHA-256已通过门禁。固定524 amu、N=100比较中，新Python
结果与原MATLAB跨求解器算法近乎相同：COMSOL/SIMION直接质量FWHM分别为
`0.0176061408/0.0196738139 Da`，R分别为`29762.34/26634.39`；标准化KDE重叠`0.86909855`、
KS距离`0.14`、配对相关`0.346157`均复现。

旧SIMION单独分析使用另一带宽`0.9*min(sigma,IQR/1.34)*N^(-1/5)`。因此同一N=5000数据迁移到
统一带宽后，524 amu直接质量FWHM由`0.02050914`变为`0.02148581 Da`，100 amu由
`0.00376789`变为`0.00408007 Da`；对应R变化只表示分析定义变化，不是轨迹或网格变化。旧数值
继续保留为迁移参考，不得与新值无标注混用。参考产物位于
`artifacts/projects/oa_tof/results/reference_analysis/baseline/`。

纯后处理迁移已闭合：Python现在直接读取SIMION`detector_crossing` TRACE，严格审计GUI Recording
的Event/PA instance/固定z面/有效半径，自动分析初始x/y/z/能量到TOF映射，并负责配对bootstrap。
原来的三个SIMION MATLAB分析器和跨求解器MATLAB峰形脚本已删除；COMSOL MPH脚本缩减并改名为
`export_fixed_particle_arrivals_from_mph.m`，只保留MPH读取、释放核对、检测面插值和CSV导出。

固定N=100、5000次配对bootstrap（seed 20260715）在统一直接KDE FWHM口径下给出的绝对R差异
2.5%/中位数/97.5%分位为`0.6273%/16.2962%/90.4652%`。旧MATLAB口径相应为
`0.7298%/17.1724%/93.0698%`，结构一致；区间很宽说明N=100不足以精确量化R差异，不能据此把
11.75%的观测差强判为确定的场差异。峰形、平均TOF和z-to-TOF映射证据仍用于定位确定性传递差异。

## 正式化门禁

候选几何只有同时满足以下条件才能转正：

1. COMSOL与SIMION共享几何契约通过。
2. 使用同源粒子比较命中率、平均TOF、直接质量FWHM、分辨率和峰形。
3. 差异有网格收敛或明确物理原因支持。
4. 正式COMSOL MPH已更新并能在GUI中重算。
5. SolidWorks零件和装配体在同一任务更新，保存错误/警告和坐标检查均通过。

## 下一步

1. 比较同一初始z的COMSOL/SIMION轴线场、单粒子轨迹及z-to-TOF映射，定位正式模型的峰肩差异。
2. 对正式COMSOL模型补做最小必要的时间步长/网格收敛检查，再决定是否值得提高COMSOL粒子数。
3. 后续制作不依赖绝对路径的SIMION同事复现包；二维轴对称COMSOL混合模型仍是未来工作。
4. **未来工作：制造与装配误差预算。** 以参数化几何为唯一来源，研究电极
   尺寸/同轴度/间隙/倾斜、绝缘件厚度、反射镜环位置和电压分配误差、检测面位置，以及材料热膨胀
   和装配基准误差对命中率、峰形和直接`FWHM_m`的影响；先做局部灵敏度，再做受公差约束的组合
   Monte Carlo，输出分辨率良率与制造公差建议。该项不得用当前候选的数值误差冒充制造误差。
