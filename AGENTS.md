# 项目执行规则（跨 Agent 共用）

本文件是 Claude Code、Codex 及其他编码 Agent 共用的执行约束。开始任务先完整阅读
`README.md`；它规定阅读顺序、知识写入位置、目录职责、生命周期、GUI 对等、CAD 同步和 Git
规则。随后只按目标项目 README 的路由读取 PROJECT 与实际使用的软件文档，不要默认通读历史或
所有软件文档。

## 必须遵守

- 修改正式项目脚本前，先阅读项目 `README.md` 和 `docs/PROJECT.md`；旧 `项目_*.md` 只在尚未完成新结构迁移的项目中暂作入口。
- 脚本分类、产物生命周期和保留条件以根README对应章节为唯一权威；Agent只额外执行本文件规定的
  删除授权、任务内归属和交接责任，不在这里维护第二份分类表。
- 删除任何文件，尤其`.mph`模型和计算结果前，必须先取得用户确认。
- 修改项目脚本时同步更新最近的项目文档；新增跨项目 COMSOL API 经验更新`docs/COMSOL_API.md`，仅跨项目验证过的策略才更新根 `docs/`。
- 新增或修改规范前，先按根README的权威表确定唯一写入位置，并用全文搜索检查是否已有同义规则；
  非权威入口只链接或简述适用范围，不复制规范正文。
- 不建立 COMSOL、SIMION、CAD 文档之间的横向网状引用；项目 README 负责导航，统一结论写入 `docs/PROJECT.md`。
- 当前会话默认使用`matlab.exe -batch`驱动的MATLAB LiveLink/Java API直连COMSOL完成服务端状态、模型加载/检查、标准操作与求解；首次使用某个直连脚本先做最小测试。若MCP恰好可用，也只作为临时检查手段，不得替代正式脚本和项目专属后处理判据。
- GUI对等与CAD同步的技术判据只由根README的“GUI 与 CAD 门禁”定义；Agent必须按变更类型执行该
  门禁，未完成适用验收时不得报告正式模型或正式几何已经完成。

## 测试报告（面向用户）

- 完成一次或多次测试后，最终交接必须给出测试报告。常规单阶段任务通常控制在**1000 tokens以内**；
  这是篇幅指引而非硬性上限。任务包含多个独立阶段、执行时间较长并产生较多需要比较的结果、存在
  失败排查或数据完整性风险、需要说明复现边界，或用户明确要求详细报告时，可以合理超过。篇幅由
  有效结论和决策信息决定，不因常规日志或重复过程增加；长任务按阶段汇总自上次交接以来的新结果，
  不重复复述已归档过程。
- 按以下紧凑顺序报告：**目标与判据**（测什么、何时算成功）；**对象与变量**（模型/样本量、对照与唯一改动）；**结果**（关键数值用表格）；**结论**（是否接受假设、适用范围）；**产物**（脚本/模型是否改动、临时产物清理、项目文档与提交）。
- 不逐条复述常规启动命令或无关日志；初始化/运行失败仅在它影响结果可信度、改变测试路径或需要复现时简述。多轮测试合并到同一张结果表，明确每轮的样本量和关键差异。
- 测试产生的项目特有结论（包括明确的负结果）仍须按本文件及`README.md`的知识归档规则写入对应项目文档；面向用户的短报告不替代项目记录。

## Git 约束

- 每次提交只包含一个可审阅主题。使用明确路径或`git add -p`暂存，不使用习惯性的`git add .`。
- 提交信息格式遵守根README的“Git规则”；Agent不得用聊天记录代替复杂提交所需的正文。
- 提交前运行`git status --short --branch`、`git diff --check`和`git diff --stat`，确认暂存区没有`.mph`、结果文件、日志或一次性脚本。
- 提交前运行`common/verify_repository_hygiene.ps1`；发现仓库根工具崩溃日志时，按根README归档到
  对应运行目录并写入manifest，不得仅因其未跟踪而忽略。
- 不覆盖、回退或夹带已有未提交改动；修改同一文件前先检查现有差异。

## Agent 自主提交

- Agent 在一个独立主题完成且已完成相称验证后，应自行创建提交，不必等待用户催促。典型时机是：一个功能/模型结论完成、一个文档规则整理完成、一个目录重构完成，或即将切换到无关主题之前。
- 若一次任务包含多个可独立审阅的主题，按主题分别提交；不要为追求更少提交而合并它们。
- 只提交本次任务产生或已明确纳入当前主题的改动。任务开始前已有的无关改动保持不动；若改动混在同一文件，先检查差异并仅在能安全拆分时提交。
- 不在结果未验证、变更范围不清楚、存在合并冲突、或用户明确要求暂不提交时自动提交。无法运行验证时，在提交说明或最终交接中写明原因。
- 本项目远程仓库是私有仓库。完成上述验证并创建提交后，Agent 应自动使用普通 `git push` 推送当前分支到其 `origin` 跟踪分支，不必逐次等待确认。
- 禁止 `--force`、`--force-with-lease`或任何远程历史改写。推送失败时保留本地提交并报告，不用替代手段绕过认证、分支保护或冲突。
- 按用户 2026-07-15 的明确决定，不增加敏感信息专项扫描；现有暂存范围、禁止二进制/结果文件和测试检查仍然有效。
